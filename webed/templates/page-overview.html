Content vs. Presentation¶

So what's actually wrong with Word or LaTex? Why consider another editor at all? Here’s why:

Word¶

Have you tried to write a book with Word or one of the other office applications? Did you enjoy writing in these word processors? Could you concentrate? Or where you constantly distracted by one of the many presentational issues? Perhaps an image was not perfectly centered, or the distance between the caption and the image was too large or too small? What about the vertical positioning on the page? And so on ...

What-you-see-is-what-you-get editors distract

All you actually wanted to do was to write the next paragraph about the subject at hand. But since it's very easy with Word to get sidetracked with minor presentational issues, the important task does not get done: Producing content! That's the problem with all those what-you-see-is-what-you-get processors: They distract and therefore lower productivity.

Admittedly, if you just want to write a simple letter complaining about your taxation, then such word processors can be very handy. But the moment you introduce elements like table of contents, footnotes, images, captions, indices, quotations, references, etc., then these programs quickly reach their boundaries.

To tackle such distractions and to enable you to write complex content, some very smart people came up with LaTex. However, that has its own share of problems. Let's investigate:

LaTex¶

If you've ever used LaTex then you know how complex and difficult its markup language is. This is because you use it not only to express the content, but also to describe the desired presentation.

Separate content from presentation

Its publication quality is truly unbeatable. However, similar to Word, Latex also mixes content production with presentational design: While Word does it on a visual level, LaTex repeats the same mistake in the markup. This is wrong, and should be fixed. There is actually a very nice example, which implements a clean separation: HTML with CSS.

HTML with CSS¶

This pair is the perfect example to demonstrate the need to separate content from presentation. While HTML focuses on the content (or structure) of a web application, CSS is the magic that turns a dull site into something that looks exciting.

But HTML with CSS has been designed only for web sites and applications inside browsers. If you ever try to print a web site it will usually look horrid! With CSS it is simply impossible to reach the same quality output of LaTex, since it has not been designed for that purpose.

ReStructuredText (rST)¶

Use rST for content

Here reStructuredText enters the arena. This incredible markup allows you to express content without worrying much about presentation. After you've written some rST, you can combine it with LaTex to produce excellent looking books, reports or articles. It's the next generation of content expression! A manuscript that has been crafted with rST is very natural to read (as compared to LaTex manuscripts or HTML code). The emphasis is on being user friendly.

But how do you combine rST with LaTex? Well, this is what NoTex does: It takes your content expressed in rST, applies a template ‒ described with LaTex ‒ and delivers auto-magically your publication ready output. If you've some LaTex knowledge then you can also edit the templates as you wish.
