Security¶

How is my communication protected?¶

NoTex applies standard SSL with 256 bit encryption, based on a certificated issued by GoDaddy.com. This means nobody between your computer and the editor's server can eavesdrop and access the data. You can confirm this statement by clicking on the padlock icon in the address bar of your browser. Plus you should see the SSL seal at the footer of the contact page: If you click on it you'll see the issuer's verification.

How is my data protected?¶

Your data is kept on servers in the European Union; the data itself is not encrypted, which means the administrators could have access to it, although internal policy forbids this. Having said that, it's planned to introduce client side encryption, in which case even a person with physical access to the hard disks would not be able to read your data.

To alleviate the current situation, we'll soon launch an offline edition of the editor. We used to offer a downloadable solution, based on a virtual machine, but due to the size of it we discontinued that product. If you're not willing to trust us or cannot afford to do so, then we recommend not to use NoTex.

Where do I sign up and login?¶

NoTex does not require you to sign up or login; just use it. If you revisit the site then you're identified based on a secure cookie which has been generated and stored (in your browser) at your last visit. If you delete your cookie then there is no possibility to associate you with the last visit; therefore the editor will create a new session for you.

Data¶

How much space do I have?¶

Not much; NoTex is not meant as a storage solution for large amounts of data. But it is planned to introduce Dropbox and Google Drive integrations. At the moment we recommended to export regularly your data to your computer.

How long do you keep my data?¶

If you do not revisit NoTex for longer than two weeks, then all your data becomes inaccessible. It is not recoverable and there is no support for any requests with respect to lost data. Therefore we recommend you to export your data on a regular basis as a ZIP archive. We do plan to address this situation rather soon by integrating NoTex with cloud storage services.

I deleted my cookies; can I access my data?¶

No. NoTex uses a secure cookie to identify revisiting users; if you delete it, then there is no possibility to associate you with your now inaccessible data. Export regularly your data to your computer, and re-import it in such a situation. Once NoTex has cloud storage integration, this problem will become irrelevant.

I deleted the sample projects; how do I recover them?¶

Performance¶

PDF conversion is slow; why?¶

Converting reStructuredText to LaTex is not an expensive operation, but translating LaTex to PDF involves significant amount of processing; even on modern machines it can take some seconds to do that.

Can the PDF conversion be accelerated?¶

Yes, to some extend: NoTex uses by default the xelatex backend to do the LaTex translation, which is compared to the pdflatex backend slower (not a lot, but the difference is noticeable). It can also happen, that our servers cannot cope with the user demand and you'll get no export at all: It such cases simply wait and retry later.

Documentation¶

There is in general not a lot of documentation available for NoTex; we're working hard to change this. You can meanwhile consult the other pages of NoTex, the Sphinx (actual engine converting rST via LaTex to PDF) documentation, the blog posts at notexeditor.blogspot.com and of course you can ask a question using the contact page.

We'll also soon launch a forum, to provide a discussion place.

What is the "options.yml" configuration file about?¶

It contains meta-information (in YAML notation) about the project and allows you to tweak the conversion parameters. This file is only relevant for rST projects.

Miscellanea¶

What does "NoTex"mean?¶

It's a pun on Tex, since if the NoTex editor is properly used, you can avoid using (and learning) Tex or LaTex. Knowing LaTex is not a disadvantage though, since you can use it to change NoTex' default templates.

Is it open source?¶

Yes it is; the source code is available at the GIT repository http://github.com/hsk81/notex and is GPL licenced. Execute git clone git://github.com/hsk81/webed.git to get a copy. The older version of NoTex is available at http://github.com/hsk81/notex.old.

What technology do you use?¶

The editor's front-end has entirely been written with the ExtJS JavaScript framework plus the JavaScript code editor CodeMirror; for the back-end the Flask Python micro-framework is being used. The back-end applies the Python document generator Sphinx to translate the rST input files to the various target formats; reStructuredText is the markup syntax and parser component of Docutils. LaTex is translated to PDF using TeX Live. The communication between the workers doing the actual conversion and the main application is managed by the ZeroMQ transport layer.
