This tutorial helps you to familiarize yourself with the user interface (UI) of the NoTex editor and guides you through your first reStructuredText (rST) project. The initial part is about the UI in general; so the impatient reader can skip forward to create his first project. New users are strongly encouraged not to skip the UI explanations.

User Interface

At the start the UI is not fully visible; follow this tutorial step by step and you'll end up with a view that looks more or less like this:

NoTex: UI Overview

On the left hand side you see the Project Manager and just below it the Document Search widgets. On top the Main Toolbar is available, below it you have an Image Viewer and two visible instances of the Text Editor. At the bottom the Status Bar offers various functionality.

On your visit, NoTex presents itself completely different: You see a a large start button in the lower central area, which is obviously used to start the editor:

NoTex: Initial UI

Please also notice, besides the Menu on top, the three little icons about notexeditor.blogspot.com, where you can find blog posts about many different aspects of NoTex, twitter.com/notexeditor, where you can follow @notexeditor, and www.facebook.com/notexeditor, where you can express how much you like it.

Hit the start button, and the editor should start: The side pane with the Project Manager will open, plus the Main Toolbar and the Status Bar will slide in:

NoTex: UI after start

The picture shows also the Document Search widget, which you can expand by clicking on the double arrows pointing upwards, visible at the bottom of the Project Manager. Once Document Search is expanded the double arrows will point downwards, allowing you to collapse the widget. Similarly, you can collapse and re-expand the Project Manager, by using the double arrows pointing sidewards, on top of that widget.

Project Manager [F9]

The Project Manager is a tree widget allowing you to navigate your reports, books, articles or what ever you're working on. Using the F9 shortcut key will expand or collapse it. The widget shows the size of your files in the right most column and aggregates them for folders and projects:

NoTex: Project Manager

Additionally, the manager has full support for different kinds of documents, based on their MIME types. If you hover the mouse over the header of a column an arrow will appear: Click on it, go to the Columns entry on the context menu, and activate the MIME column.

Document Search [F10]

If you haven't already done so yet, expand the Document Search widget. Using the F10 shortcut key you can always quickly focus the cursor in the search field: Enter a path or file name (or parts of it) you're looking for, and it will filter the list and show you the corresponding results:

NoTex: Document Search

In addition to simple text search, it supports also regular expressions: E.g. if you're looking for all documents over all your projects containing txt in its filename then do a search simply for txt, or if your looking for images ending with the jpg extension then do a search for \.jpg$. If you want to list all files in the Article project, do a search for Article/, or if you look for all JPEG images within the Article project then the Article/(.*)\.jpg$ search will do.

Regular expressions are complex beasts; if you're not comfortable using them, just stick to simple text search: In most cases the latter will be more than enough.

Text Editor

In the Project Manager open Article (either with a single click on the plus sign next to the project's icon, or with a double click on the project's title); then select the content.txt file: A Text Editor tab will be created, allowing you to edit the content.

The Text Editor has it's own toolbar, which is specialized for the kind of text (or markup) you're working on. Since the current document is based on rST, you see the toolbar with the corresponding buttons:

NoTex: Text Editor (with a toolbar specialized for rST)

If the kind of text (or markup) does not support a specialized toolbar, then it will at least have a generic one, offering you basic functions like undo, redo, cut, copy, paste, case conversion, indentation, split view plus search and replace functions:

NoTex: Generic Toolbar

Now, on the toolbar for rST click the Horizontal Split button: Another tab will be created containing the same document Article/content.rst. Both tabs are linked, so any change you make in one will immediately be reflected in the other. Then, open the Article/options.yml configuration: A new tab will be created with the corresponding content. Click on associated toolbar the Vertical Split button:

NoTex: Splitted Text Editors

The split views allow you to work on different parts of a document at the same time; or you can work with or compare different documents next to each other without the hassle of changing tabs all the time.

Image Viewer

Open the Article/resource/emcc.jpg file: An Image Viewer instance within its own tab will be created, displaying the image. NoTex is able to show all images for which your browser has the corresponding capability. The Image Viewer does not have a toolbar, so if you want to display an image and work on a text document next to the image, then you have to first open the text document, split it and then open the image file.

Status Bar

If you've noticed, in the Article/options.yml document some lines wrap at the right edge of the editor, because they are too long. For such cases NoTex allows you to zoom out the document, using the slider on the Status Bar:

NoTex: Status Bar

The Status Bar shows you a message if something goes wrong, or if something is in progress (e.g. an export operation). It has a button that displays the current line and character position (should your cursor be inside a text document): If you click on it, the button will count the number of lines, words and characters for you. Further, you can change the spell check language:

NoTex: Spell Check Languages

NoTex supports more than 85 language and country pairs. If you select an entry for the first time, it downloads the corresponding dictionary and processes it to enable fast spell checking. With some languages (e.g. Turkish) where the dictionary corpus is large this may take some time. For a dictionary, which has already been downloaded before, the overall time to enable spell checking will be shorter. To disable it just delete the current language, and don't provide any input.

On the right most position of the Status Bar, you can change the text zoom. The little panel showing the current zoom factor is a button, and by clicking on it you'll reset the current value to 100%. This setting is used only for the display, and does not influence any other functionality (like the font size in PDF exports)!

Main Toolbar

The Main Toolbar allows you to work with documents, manage projects (and documents), offers ZIP archiving and export as functionality:

NoTex: Main Toolbar

NoTex does not auto-save, but the title of a document indicates if it needs to be saved or not: An italic title shows a document has been edited but not saved yet. Opening a document means uploading a single file from your local machine into the currently selected project. You can create new projects, folders or documents, rename or delete them. You can import a project from a ZIP archive, which has previously been exported as such. Further, you can export projects as PDF, HTML, or LaTex. For now, the latter three exports are available only for rST projects.
